# Understanding Next.js: App Router, Server Components, and Data Fetching

## Overview
Next.js is a React framework that enables server-side rendering, static site generation, and hybrid rendering. With the introduction of the App Router (based on React Server Components), Next.js 13+ offers a paradigm shift in data fetching, layout, and streaming.

## Why It Matters
- Improves performance by shifting rendering to the server.
- Reduces client-side JavaScript bundle size.
- Simplifies data fetching with async Server Components.
- Enables incremental static regeneration (ISR) and edge computing.

## Real-world Usage
- E-commerce platforms (e.g., Shopify storefronts) use Next.js for SEO and fast product pages.
- SaaS dashboards leverage Server Components for authenticated data fetching without exposing API keys.
- Marketing sites benefit from static generation and ISR for up-to-date content.

## Code Example
```jsx
// app/dashboard/page.tsx
export default async function DashboardPage() {
  // Fetch data directly in a Server Component
  const res = await fetch('https://api.example.com/dashboard', {
    headers: { Authorization: `Bearer ${process.env.API_TOKEN}` }
  });
  const data = await res.json();

  return (
    <section>
      <h1>Dashboard</h1>
      <DataGrid data={data.metrics} />
    </section>
  );
}
```

## Common Mistakes
- Using `useEffect` for data fetching in Server Components (not needed).
- Forgetting to mark a component as `async` when awaiting data.
- Overusing client-side state when server-side props would suffice.
- Misplacing `'use client'` directive, causing unnecessary client bundles.

## Interview Question
**Q:** How does Next.js 13 App Router differ from the legacy Pages Router in terms of data fetching and rendering?  
**A:** The App Router uses React Server Components by default, allowing asynchronous data fetching directly in components without `getStaticProps` or `getServerSideProps`. It supports streaming, nested layouts, and co-located UI with data fetching logic, while the Pages Router relies on exported async functions and client-side React lifecycle methods.

## Key Takeaways
- Next.js App Router shifts data fetching to the server, reducing client bundle.
- Server Components are async and can fetch secrets safely.
- Use `'use client'` only when interactivity or hooks are needed.
- Layouts and nesting are first-class concepts in the App Router.
- Hybrid rendering (SSG, SSR, ISR) is still available via route segment config.
